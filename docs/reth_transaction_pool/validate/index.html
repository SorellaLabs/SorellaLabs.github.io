Constants

  • MAX_CODE_SIZE
    Validation constants. Maximum bytecode to permit for a contract
  • MAX_INIT_CODE_SIZE
    Validation constants. Maximum initcode to permit in a creation transaction and create instructions
  • TX_MAX_SIZE
    Validation constants. TX_MAX_SIZE is the maximum size a single transaction can have. This field has non-trivial consequences: larger transactions are significantly harder and more expensive to propagate; larger transactions also take more resources to validate whether they fit into the pool or not.
  • TX_SLOT_SIZE
    Validation constants. TX_SLOT_SIZE is used to calculate how many data slots a single transaction takes up based on its size. The slots are used as DoS protection, ensuring that validating a new transaction remains a constant operation (in reality O(maxslots), where max slots are 4 currently).
