fn on_canonical_state_change(&self, update: CanonicalStateUpdate<'_>)

Event listener for when the pool needs to be updated

Implementers need to update the pool accordingly. For example the base fee of the pending block is determined after a block is mined which affects the dynamic fee requirement of pending transactions in the pool.
